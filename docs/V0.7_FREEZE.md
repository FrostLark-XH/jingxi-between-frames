# 镜隙之间 V0.7 Freeze

## 版本定位

V0.7 是"重新显影版"。

本版本在 V0.6"真实显影引擎版"的基础上，完成 AI 字段收口、重新显影闭环、后台 AI 存档不阻塞三大核心能力。

## 核心能力

1. **原文可编辑**：FrameDetailOverlay 支持编辑已保存的原文
2. **AI 结果可判断过期**：`contentHash()` 检测原文变化，`isAiStale()` 标记过期摘要
3. **用户可手动重新显影**：编辑原文后一键重新调用 AI，失败保留旧摘要和标签

## 完成项

### 1. 重新显影能力
- 编辑原文后 FrameDetailOverlay 检测 `contentHash` 变化
- "重新显影"按钮可一键重新调用 AI
- 重新显影失败时保留旧摘要和标签，不丢失数据
- 只改标签不触发 stale 判断

### 2. 存档不阻塞
- 新帧立即保存，AI 在后台静默更新
- 用户保存后帧立即可见于时间胶片
- AI 通过 `onUpdateFrame` 异步更新 summary/tags/tone
- 等待态显示"请静候时光沉淀…"

### 3. AI 字段收口
- 单帧输出：`{ summary, tags (2-3个), tone }`，去掉 keywords
- 日卷输出：`{ mainline, themes (3-5个), reviewHint }`，keywords → themes
- `MemoryFrame.keywords?: string[]` 保留仅向后兼容
- Prompt 角色："暗房显影师 / 轻文学整理者"

### 4. Prompt 风格校准
- summary 长度硬约束：24-48 中文，尽量 28-42
- 每条最多一个轻微意象，禁止连续堆叠画面
- 禁止过度拟人化、散文腔、古风腔
- 禁止重词：灵魂、命运、宇宙、救赎、永恒、深处、迷雾
- tone 判断规则：按文字气息而非内容推测

### 5. 附属修复
- 保存时自动调用 `formatText()` 为每段添加全角空格首行缩进
- 修复 `String.trim()` 误删 `　` 的 bug
- DaySummaryCard 显示"这一卷的主题"

### 6. Provider 兼容
- 支持 DMXAPI（raw key 鉴权）和 SiliconFlow（Bearer 鉴权）
- `LLM_PROVIDER` 显式设置或 URL 域名自动检测
- 三层 fallback：realProvider → mockProvider → client-side 降级

### 7. 验收结果
- `npm run build` 通过
- `tsc --noEmit` 零错误
- 5 条测试文本 AI 输出质量验收通过
- 重新显影完整闭环验收通过
- Provider: DMXAPI，Model: `deepseek-chat` → `deepseek-v4-flash`

## 当前边界

- 无语音输入
- 无 PDF / 图片导出
- 无 AI 聊天
- 无心理分析
- 无账号系统
- 无数据库
- 无云同步
- 数据仍保存在 localStorage

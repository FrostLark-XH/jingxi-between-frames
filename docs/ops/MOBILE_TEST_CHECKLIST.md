# 镜隙之间 手机测试清单

> V0.9 PWA 实验版真机验证清单。

## 测试环境

- 线上地址：Vercel 部署后的 `.vercel.app` 域名
- 测试设备：iPhone (Safari) + Android (Chrome) 各一台
- 网络：Wi-Fi + 蜂窝数据各测一轮

## 基础功能

- [ ] 1. iPhone Safari 打开线上地址，页面正常加载
- [ ] 2. "添加到主屏幕"选项可用
- [ ] 3. 从桌面图标打开，以 standalone 模式运行（无浏览器工具栏）
- [ ] 4. textarea 输入一帧文字
- [ ] 5. 点击"开始显影"，按钮状态变化，帧保存
- [ ] 6. AI 摘要 / 标签 / tone 在 1–3 秒内返回
- [ ] 7. 点击"查看时间胶片"，帧卡片正常展示
- [ ] 8. 点击帧卡片打开详情浮层
- [ ] 9. 编辑原文 → 重新显影 → AI 摘要更新
- [ ] 10. 详情浮层 → PNG 导出 → 图片保存到相册
- [ ] 11. 档案箱 → 勾选多帧 → 导出合集长图 PNG
- [ ] 12. 刷新页面后，之前保存的帧仍在（localStorage）
- [ ] 13. 切换主题（3 套），颜色正确
- [ ] 14. 键盘弹出时不遮挡底部按钮和输入区
- [ ] 15. safe-area 底部（iOS Home Indicator）不被内容遮挡
- [ ] 16. 横屏模式下界面可用（不要求完美适配）

## 网络异常

- [ ] 17. 飞行模式下打开页面，已有帧正常显示
- [ ] 18. 飞行模式下保存帧，提示"网络不可用"或静默 fallback 到 mock
- [ ] 19. 从飞行模式恢复后，新建帧可正常显影

## Android 专项

- [ ] 20. Chrome 打开 → 添加到主屏幕 → standalone 模式正常
- [ ] 21. 返回键不退出 App（standalone 内正常导航）
- [ ] 22. 通知权限不弹窗（当前阶段不需要通知）

## 已知限制

- 暂不做离线缓存（Service Worker）
- 暂不做后台同步
- 数据仅存 localStorage，清除浏览器数据会丢失所有帧
- Android 部分设备可能有键盘遮挡问题（依赖系统 WebView 版本）

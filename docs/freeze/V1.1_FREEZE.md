# 镜隙之间 V1.1 Freeze

## 版本定位

V1.1 是"App 预览稳定版"。

在 V1.0 内测安全版基础上，重点修正本地数据一致性、移动端菜单层级、E2E 稳定性，以及"镜中人"入口与页面气质。目标不是加功能，而是让当前体验更适合继续作为 App 原型预览。

## 核心完成项

### 1. 数据一致性修复

- `hooks/useAppState.ts`
  - 编辑原文后同步更新 `rawContent` / `preview` / `wordCount`
  - 原文变更后将 AI 状态标记为 `stale`
  - AI 回写后同步 `developStatus = developed / failed`
  - 删除 / 恢复时同步 `frameStatus = deleted / active`
  - 永久删除前自动创建备份
  - localStorage 数据改为挂载后加载，修复首屏 hydration mismatch

### 2. 数据管理备份修复

- `components/DataManager.tsx`
  - 备份导出改为完整 `MemoryFrame` 结构，不再遗漏 v3 字段
  - 导入备份时走 `migrateAll()` 归一化，兼容旧数据结构

### 3. 移动端 Header / More 菜单优化

- `components/AppHeader.tsx`
  - More 菜单改为暗房质感浮层，避免突兀浅色卡片
  - Header 层级高于全局右上控制区，避免触控遮挡
- `components/ArchivePanel.tsx`
  - 移动端 More 菜单中的"档案"保留文字，不再只显示孤立图标
- `app/page.tsx`
  - 子页面隐藏全局主题按钮，减少移动端右上角操作冲突

### 4. 时间胶片页镜中人入口收束

- `components/FilmPage.tsx`
  - "镜中似乎映出了些什么 →"从 Header 命令区移到年月日切换下方
  - 入口改为淡线 + 低透明文字，不再像菜单项或贴片
  - 少于 3 帧时显示弱提示："再多几帧，镜中会浮出些什么"

### 5. 镜中人页面视觉收束

- `components/ReflectionPage.tsx`
  - 去掉上一版过重的碎片爆开效果
  - 保留少量边缘词片、细线与微尘
  - 碎片默认缓慢下坠 / 漂移，形成轻微重力感
  - 鼠标或触摸经过空白区域时，附近碎片被弱重力点轻微吸引并显影
  - 正文卡片层级保持最高，阅读区域不参与碎片交互
  - 装饰碎片 `aria-hidden`，不进入阅读顺序

### 6. E2E 稳定性

- `tests/e2e.cjs`
  - 支持 `E2E_BASE_URL`
  - 修正移动端 More 菜单打开路径
  - 修正 JSON BOM 解析
  - 草稿 debounce 等待时间加余量
  - 抽屉内"全选"改为 DOM 点击，避免 Playwright 视口判定误伤

## 已知提示

- `metadataBase` 未设置，构建时仍会提示 Open Graph / Twitter 图片解析回退到 localhost
- Edge runtime 会提示影响静态生成
- 3000 端口存在旧 `node/workerd` 进程占用，Windows 拒绝当前会话停止；开发预览改用 3001

## 当前边界

- 无账号系统、无云同步、无后端数据库
- 数据仍保存在 localStorage
- 暂不处理应用商店上架
- 暂不处理中国大陆访问稳定性
- 暂不做 PDF 导出
- 暂不做语音输入
- 暂不做 AI 聊天 / 心理分析

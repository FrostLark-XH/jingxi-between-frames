## V0.4 新增内容（2026-04-29）

### 移动端输入体验修正
- **快捷保存条**：移动端 `fixed bottom-0` sticky bar，复用 ActionBar 设计（`h-12` 全宽渐变按钮），`env(safe-area-inset-bottom)` 避让安全区
- **取消自动弹键盘**：移动端不自动 focus textarea，桌面端保留
- **点击外部收起键盘**：点击 textarea 和保存条以外区域 blur
- **草稿 localStorage 持久化**：key `jingxi_draft`，300ms debounce 写入，页面切换/刷新不丢文字
- **误触修正**："查看时间胶片" 入口在有文字或聚焦时隐藏

### 数据导出
- 档案箱改为**逐帧显示 / 逐帧勾选**（替代 V0.3 的日期级勾选）
- 支持 JSON / Markdown / TXT 三种格式导出
- TXT 导出自动加 UTF-8 BOM，所有格式 `charset=utf-8`

### 移动端保存条修正（2026-04-29 第二轮）
- 移除 `isFocused` 门控：保存条在键盘收起时不再消失

### 性能优化
- **TimelineRail**：`useSpring` + `useTransform` 替代 `useState` 驱动光球，零 React 重渲染
- **MemoryCard**：`React.memo` + 自定义 `arePropsEqual`，跳过不变帧渲染
- **移动端**：`whileHover` 仅在桌面端启用，blur 从 3px 降至 1px

### 时间导轨优化

  - 显影中：沿输入框底部边缘向左扫描，伴随镜隙高光线
  - 时间胶片：作为时间导轨当前节点，新帧到达时播放 arrival 动画
- 时间标签碰撞检测：间距 < 24px 跳过，仅显示首尾 + 不重叠标签
- Spring 参数调整为 stiffness: 60 / damping: 28 / mass: 0.5（更慢更柔）
- 帧节点标记缩至 2.5px，opacity 降至 0.45
- STOP_WORDS 从 ~47 扩充至 ~70
- 新增 13 条 TOPIC_RULES 正则（时间胶片、产品设计、编程、猫等）
- THEME_PATTERNS 修正：移除单字匹配，修复重复分类

## 一、完成内容

### 产品架构
- Next.js 15 App Router + React 19 + TypeScript
- Tailwind CSS v3 + Framer Motion
- useReducer 全局状态 + localStorage 持久化
- 移动端优先，max-w-[480px] 单列布局

### 核心页面（2 个）

**记录室首页**
- 输入框（35vh 最小高度，暖色左边框 `border-warm`，内发光 `accent-glow`）
- 「开始显影」保存按钮（accent→accent-soft 渐变，激活态微光）
- 显影动画：保存时文字模糊→清晰过渡（500ms CSS blur + overlay pulse）
- 状态栏：首次「准备留下第一帧」，保存后「下一帧 · Frame NNN」
- Toast 保存反馈（暖色边框，无绿色成功色）
- 底部入口「查看时间胶片 · 今日 N 帧」

**时间胶片页**
- 日/月/年/片段四尺度切换（按帧数渐进解锁：<10 帧仅日/片段，<30 加月，≥30 加年）
- 卷首卡（DaySummaryCard）：日期 + 帧数 + 关键词
- 记忆卡片（MemoryCard）：「第 XX 帧」格式，时间/原文/AI 摘要/标签/元数据
- 详情浮层（FrameDetailOverlay）：全文/摘要/标签/复制/软删除
- 时间导轨（TimelineRail）：仅展示，滚动跟随光点 + 帧节点标记 + 镜隙高光
- 空状态引导页

### 数据与状态
- MemoryFrame 完整模型：content / summary / tags / status / createdAt / deletedAt
- 软删除 + 7 天自动清除
- 回收站面板：恢复 / 永久删除（仅在有已删除帧时显示入口）
- 旧数据自动迁移（loadFrames 时转换旧格式）

### AI 模拟层
- 规则摘要生成：识别问句/短句/长文本模式，≤30 字
- 规则标签生成：二元分词 + 停用词过滤 + 情绪词典 + 主题正则，≤5 标签

### 主题颜色系统
- 3 套主题：雾灰暗房（默认）/ 暮色豆沙 / 晨雾米灰
- 7 个基础 CSS 变量 + 9 条 `color-mix()` 派生规则
- 主题切换后所有表面/边框/文字自动染色
- localStorage 持久化（key: `jingxi_theme`）
- Three.js WebGL shader 背景随主题联动
- 右上角调色盘切换按钮（极淡，约 11% 有效透明度）

### 组件清单（17 个）
`AppShell` / `ShaderBackground` / `RecordingRoom` / `MemoryInput` / `ActionBar` / `StatusToast` / `FilmPage` / `MemoryTimeline` / `MemoryCard` / `MemoryCardSkeleton` / `DaySummaryCard` / `FrameDetailOverlay` / `DevelopingDot` / `TimelineRail` / `TimeScaleSwitcher` / `EmptyState` / `RecycleBin` / `ThemeSwitcher`

## 三、已冻结的核心体验

1. **记录 → 显影 → 归档**：输入文字 → 点击「开始显影」→ 短暂模糊动画 → 保存为帧
2. **时间胶片浏览**：日视图查看当天记录，卡片按时间排列，点击查看详情
3. **帧系统**：每帧带编号「第 XX 帧」、AI 摘要、标签、字数、状态
4. **软删除与回收站**：删除移至回收站，7 天自动清除，可恢复
5. **主题切换**：3 套主题，切换后全局统一染色，刷新保持
6. **首页优先输入**：无 Dashboard、无统计图表、无管理面板

## 五、已知风险

| 风险 | 影响 | 备注 |
|------|------|------|
| localStorage 容量限制 | 大量帧可能超出 5-10MB 配额 | 当前无后端，数据仅存本地 |
| 隐私浏览模式 | localStorage 可能在隐私模式下不可用或会话结束后清除 | 已 try-catch 保护 |
| 移动端 Framer Motion 性能 | 卡片 layout 动画在中低端机上可能掉帧 | 未实测 |
| 时间导轨仅展示 | 帧数 >20 时无快速定位手段 | 当前为 P0 修正结果，后续可恢复拖拽 |
| `color-mix()` 浏览器兼容 | Safari 16.2+ / Chrome 111+ 支持，旧设备可能看不到表面染色 | 有 `:root` fallback 基础色 |
| 保存链路单线程 | `isDeveloping` guard 阻止连存，快速连续保存会丢后面的输入 | 设计如此，防止重复提交 |
| 无数据备份 | 清除浏览器数据会丢失所有记录 | 已有 JSON/MD/TXT 导出可供手动备份 |

## 六、下一版本建议方向

按优先级排序：

1. **真实 AI 接入**：摘要和标签从规则模拟升级为大模型生成。这是产品核心差异化。
2. **语音输入**：Web Speech API 或录音 → 转文字，补上首页缺失的输入方式。
3. **PDF / 图片卡片导出**：在现有 JSON/MD/TXT 导出基础上增加美化格式。
4. **时间导轨恢复拖拽**：当帧数积累到 20+ 后，仅展示的导轨实用性不足。
5. **胶片隐喻视觉强化**：当前「一天是一卷胶片」的概念还不够视觉化。
6. **移动端更多输入优化**：textarea 高度自适应、iOS accessoryView、草稿保存指示器等。
